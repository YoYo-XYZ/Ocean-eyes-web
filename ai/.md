The current JSON script will not work as-is for a web app, but it's a great starting point. Here's why and what's needed:
What works well:

- JSON output format is perfect for REST API responses
- ONNX Runtime is production-ready for server deployment
- The inference pipeline itself is solid
  What's missing for web use:
  Issue	Current Script	Web App Requirement
  Input	File path argument (path/to/image.jpg)	Receives image data (bytes/base64/multipart form)
  Model loading	Loads 3 models on every invocation (3-5 seconds)	Load once at startup, reuse for all requests
  Concurrency	Single-threaded CLI	Handle multiple concurrent requests
  Transport	stdout/file output	HTTP response with proper headers
  Architecture	Standalone script	Needs HTTP server wrapper (FastAPI/Flask)
  What you'd need to add:

1. Web framework — FastAPI (recommended) or Flask
2. Model caching — Load ONNX sessions once at startup, not per-request
3. Image upload handling — Accept multipart/form-data or base64
4. Async support — Use async endpoints for non-blocking I/O
   Example architecture:
   ┌─────────────┐      ┌──────────────┐      ┌─────────────────┐
   │   Client    │ ───▶ │  FastAPI     │ ───▶ │ ONNX Inference  │
   │  (Flutter)  │      │  /predict    │      │  (cached models)│
   └─────────────┘      └──────────────┘      └─────────────────┘
   │
   ▼
   ┌──────────────┐
   │ JSON Response│
   └──────────────┘
   Recommendation: Create a scripts/utils/api_server.py that wraps the inference logic in a FastAPI app. The existing run_pipeline_onnx_json.py logic can be extracted into reusable functions.
